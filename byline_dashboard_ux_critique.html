Current navbar — critical problems
The current top navbar is a generic web header. No depth, no power user shortcuts, no command palette. It doesn't feel like a tool you live in daily.
Miss No command palette
Every developer tool in 2025 has ⌘K. Byline has none. This is the single highest-leverage addition — lets power users publish without touching the mouse.
Miss Flat sidebar, no hierarchy
Current agent config + recent bylines dump everything at the same level. No grouping by project, no separation between pipeline controls and history.
Miss No status bar / system tray
The footer just shows navigation links. It should be a persistent status bar showing agent health, last run time, Composio connection status, and pipeline queue.
Miss No project switcher
A user running Byline for 4 projects has no way to filter their feed to a single project. The entire app needs a project context, like Vercel's team switcher.
Recommended sidebar structure — inspired by Linear
Two-level sidebar: top-level for workspace globals, second level for project-scoped views. Collapsible to icon-only mode (like Warp).

Project context cards Missing
Each project (fltrd.tech, Miryn, etc.) gets its own card in the sidebar and a context page: tech stack, tagline, current milestone, metrics, GitHub link, and an "update context" button. Every draft pulls from this — it's the core memory layer. Right now you have zero of this.

Voice profile page Missing
Let users paste 5-10 of their old LinkedIn/X posts. Run them through an extractor that builds a profile: avg paragraph length, sentence openers you never use, words you overuse, your tonality score. Display it visually. Show it as a diff against the current draft. This is the biggest differentiator from every other tool.
Design this ↗
03
Draft review / "The Desk" view Partial
The demo on the landing page shows this beautifully — 4 platform tabs, edit/approve buttons, quality score. But the actual dashboard doesn't have it. The "Recent bylines" table is not the same thing. You need the full desk view as the primary dashboard action, not buried in a demo.
04
Agent pipeline run log / trace view Missing
When a milestone runs through the pipeline, users should be able to click "view run" and see each agent's output, token usage, time taken, and any flags. Like Vercel's build log. This builds trust in the AI and lets users debug bad outputs.
Explore this ↗
05
GitHub integration page Missing
The landing page mentions GitHub webhook ingestion — auto-detect releases and significant PRs. But there's no UI to set this up. You need a page: connect repo → choose trigger rules (releases only? PRs above N lines? Tags?) → preview what gets turned into bylines.
Priority 2 — strong differentiators
06
Platform performance analytics Missing
The dashboard shows a "posts / week" bar chart and a voice score trend. That's it. You need: engagement by platform, top-performing post types, best days/times to post, and a feedback loop that feeds back into agent prompts. Like a personal social media analytics layer.
07
Milestone queue / content calendar Missing
A lightweight calendar or queue view showing: upcoming scheduled posts, milestones logged but not yet published, and suggested optimal posting times per platform. Ghost and Beehiiv do this well. It makes the tool feel like an OS, not a one-shot generator.
08
Narrative arc manager Missing
The "build in public" narrative arc is mentioned in the demo dropdown, but it's just a selector. Build a full arc manager: create a story arc (e.g. "zero to 1000 users"), add chapters, tag milestones to chapters. The arc gives every post coherence — post 7 of the 1000-user arc feels different from a standalone update.
Explore this ↗
09
Cross-project synthesis view Missing
From the original spec: if you've built 4 projects all using pgvector, Byline should spot the pattern and suggest a synthesis post — "Lessons from 4 RAG projects." No other tool does this. It requires pgvector semantic search across all your logged milestones.
10
Voice note / quick capture widget Partial
The CLI has `byline log "..."`. The dashboard has a text area. But there's no voice note button (Whisper) in the dashboard — it's mentioned in the features section but not actually built. Add a mic button next to the text area that records → transcribes → submits instantly.
Priority 3 — polish and depth
11
Build log / changelog feed Missing
A chronological public-facing feed of all your logged milestones (not the posts, the raw milestones). Like an internal changelog. Can be optionally public at byline.local/changelog. Founders love this as a second artifact.
12
Subreddit intelligence panel Missing
For Reddit posts — show which subreddits the strategist agent considered, why it picked one, what the top posts in that community look like, and the anti-promo score. Makes the "Reddit Stealth" feature tangible instead of a black box toggle.
13
Custom post templates / styles Missing
Let users save their own reusable formats: "problem → solution → lesson" for LinkedIn, "What I shipped / what broke / what I learned" for X threads. The agent uses these as starting structures rather than generating from scratch every time.
14
Team / collaborator view Missing
The cloud plan mentions "team workspace" — but there's no UI for it even in prototype form. Add at least a placeholder page showing how multi-user Byline would work (review queue, draft comments, approval flow). Signals the roadmap clearly.

CLI-first vs dashboard-first — my recommendation
Go CLI-first for input, dashboard-first for review. The split is clear:

CLI does: log milestone, trigger pipeline, check status, quick approve. Developers want this in their terminal next to their code. `byline log "..."`, `byline status`, `byline approve [id]`, `byline draft [id] --platform linkedin`.

Dashboard does: review drafts side-by-side, edit with formatting, view analytics, configure voice profile, manage projects, see agent trace. The visual judgment required for draft review needs a full UI.

The dashboard should also have a sticky input bar at the bottom (like Claude's chat bar) so keyboard-first users never need to scroll to log something — but the CLI is the primary for that action.
User control — current state is minimal
The current dashboard has platform toggles, voice strength slider, critic score floor, posting frequency (low/medium/high), and 5 agent skill toggles. That's a start, but a Founder OS needs much deeper customization. Here's what to add.
Per-platform post templates
Let users define their own post structures per platform. A LinkedIn template might be: hook line → 3 bullet lessons → CTA. An X template: thread opener → 4 thread posts → landing tweet. The agent fills the structure, not invents it.
Tone dial per project
Different projects might warrant different voices. fltrd.tech posts might be more technical; a side-project might be more casual. Let users set per-project tone overrides rather than one global voice profile.
Agent skill marketplace
The "Skills" concept in the doc is the right idea. Tier 1 skills are built-in. Tier 2 can be community-contributed skill modules (a "YC voice" skill, a "technical deep-dive" skill, an "indie hacker" Reddit writer). Ship this as the open-source extension point.
Keyword / topic blocklist
Let users specify words or topics to never mention (competitors, sensitive business context, under-NDA tech). The critic agent checks drafts against this list before approval. Simple but high-trust signal.
Approval modes
Three modes: Manual (always review before posting), Semi-auto (auto-post if critic score ≥ threshold), Auto (post everything that passes critic). Make this per-platform. Auto for Threads (low-stakes), manual for LinkedIn (high-stakes).
Prompt inspector / override
Let power users see and edit the actual prompts sent to each agent. Raycast-style "edit this action" mode. Makes the tool feel transparent and hackable — critical for developer adoption.
Enhanced agent config UI
The current agent config is a set of toggles. It needs sub-pages per agent with detailed controls.
Strategist
Platform weights, post-worthiness threshold, angle preferences
LinkedIn writer
Paragraph length, hook style, CTA behavior, emoji policy
Reddit writer
Subreddit list, anti-promo intensity, flair preferences
X writer
Thread length, hashtag policy, reply hook style
Critic
Score weights, rejection reasons, custom checks
Scheduling
Best time to post per platform, timezone, frequency cap

Avatar system
Use Boring Avatars (open source) for project avatars. Generated from project name hash — deterministic, beautiful, unique. Each project gets its own color set. Shown in sidebar, project cards, and draft headers.
import Avatar from 'boring-avatars' <Avatar name="fltrd.tech" variant="marble" colors={["#E8593C","#2C2C2A"]} />
Icon library
Use Tabler Icons (7000+ outline icons, MIT license) for all UI icons. It's already used in some places. Replace any emoji or inconsistent icons with Tabler throughout the dashboard. The outline style matches the minimal aesthetic exactly.
npm install @tabler/icons-react import { IconRocket, IconBrandLinkedin, IconTerminal2 } from '@tabler/icons-react'
Micro-animations
Three specific animations that make the product feel alive: (1) Pipeline progress — animated dashed line between agent nodes that fills solid on completion. (2) Critic score reveal — counter animates up to the score on load. (3) Draft approval — subtle confetti or checkmark pulse when you click Approve & Ship.
// Use framer-motion import { motion } from 'framer-motion' // Score counter animation const count = useMotionValue(0) const display = useTransform(count, v => v.toFixed(1))
Terminal-style log viewer
Agent run traces should look like a terminal (dark bg, monospace font, green/yellow/red log levels). Use xterm.js or just a styled scrolling div. This is the visual language of developer tools — it signals "this is built for you."
Specific CSS improvements for the dashboard
Five concrete changes that have the biggest visual impact:
1
Add a thin left accent stripe to active sidebar items
Like Linear. 2px solid primary color, flush left. Makes active state unmistakable without heavy backgrounds.
2
Replace all full-opacity backgrounds on status badges
"Posted" = green-100 bg + green-700 text. "Draft" = amber-100 + amber-700. "Flagged" = red-100 + red-700. Never full green/red background — too aggressive.
3
Add monospace font to all scores, metrics, and dates
Critic scores (8.6), platform counts (47), timestamps — use `font-family: var(--font-mono)`. Makes numbers feel precise and developer-native.
4
Improve the KPI card grid
Current cards (Posts published, Avg critic score, etc.) have no trend context. Add a tiny sparkline or ↑↓ percentage delta below each number. 2 lines of data per card tells a much richer story.
5
Dark/light mode toggle in the header
Already in the landing page nav (the moon icon). The dashboard also has it. Make sure it's persistent via localStorage and respects system preference on first load.
Lovable / Cursor prompts to fix everything
Structured prompts you can paste directly into Lovable or Cursor. Each prompt is self-contained and builds on the existing codebase. Start with #1 and go in order — they build on each other.
Prompt 01 — sidebar redesign
Redesign the dashboard sidebar to use a two-level hierarchy. Add a workspace section at the top with a Byline logo + "self-hosted" badge. Below that, add three navigation groups: "Workspace" (Overview, Analytics, Feed), "Projects" (one item per project from the database, with a boring-avatar icon, plus an "Add project" button at the bottom), and "Settings" (Voice profile, Integrations, CLI & API). Make the active item have a 2px left border in the primary accent color (#E8593C) with a subtle background. Add an icon-collapse toggle at the bottom to hide labels and show icon-only mode. Use Tabler Icons for all nav icons.
Prompt 02 — status bar footer
Replace the current footer with a persistent bottom status bar (height 36px, fixed to bottom of the sidebar/main area). Show 4 status pills: Composio connection (green check if connected, red X if not), LangGraph health (green with response time in ms), last pipeline run (relative timestamp like "3m ago"), and queue count ("2 pending" or "idle"). On the far right, show the app version (from package.json) and a keyboard shortcut hint for the command palette (⌘K). This bar should always be visible but never interrupt the main content flow.
Prompt 03 — command palette (⌘K)
Add a command palette triggered by ⌘K (or Ctrl+K on Windows). It should overlay the entire dashboard. Include these commands grouped by category: "Actions" — Log new milestone, Run pipeline, Approve last draft; "Navigate" — Go to Overview, Go to Analytics, Go to Voice Profile, Open project [name]; "Agent skills" — Toggle Storyteller mode, Toggle Reddit Stealth; "Settings" — Open integrations, Edit .env. Use a fuzzy search filter at the top. Animate the palette in/out with a subtle scale + opacity transition. Dismiss on Escape or click outside. Style it like Linear's command menu — dark background, monospace font for shortcuts, grouped sections with labels.
Prompt 04 — project context pages
Create a project detail page reachable from the sidebar. It should have: a header with the project name, a boring-avatar (use the boring-avatars npm package with variant="marble"), the project tagline, and a GitHub link. Below that, two columns: left column shows "Project context" — an editable card with fields for: description (textarea), tech stack (tag input), current milestone (text), and key metrics (key-value pairs the user can add/edit). Right column shows "Recent bylines" — last 5 milestones logged for this project, each with platform icons, score, and status. Add a "Update context" button that saves changes and triggers a re-embedding of the context in pgvector.
Prompt 05 — voice profile page
Create a Voice Profile page at /voice. It should have three sections: (1) "Train your voice" — a large textarea where users paste 5+ past posts (LinkedIn, X, or Reddit), with a "Analyze posts" button that calls the backend to extract the voice profile. (2) "Your voice fingerprint" — display the extracted profile as a set of visual metrics: avg sentence length (shown as a bar), paragraph density, vocabulary richness, formality score, and a list of phrases you overuse (as tags) and phrases you never use. (3) "Voice check" — a text input where users can paste any draft and get a voice match score + specific flags ("This sentence doesn't sound like you"). Show a before/after diff visualization.
Prompt 06 — first-run onboarding
Add a first-run onboarding wizard that shows when a user has no projects and no voice profile configured. It should be a full-screen modal overlay (not a separate page) with 5 steps: (1) Name your first project — name field, description, tech stack tags, GitHub URL. (2) Voice training — paste 3+ past posts. (3) Connect platforms — OAuth buttons for LinkedIn, X, Reddit, Threads via Composio (show each platform as a card, green check when connected). (4) Log first milestone — a text area with placeholder "What did you ship today?" and a Run Pipeline button that shows the agents executing in real time with step indicators. (5) Review and ship — show the LinkedIn draft with approve/edit buttons. Include a progress bar at the top showing current step. Allow users to skip steps.
Prompt 07 — agent run trace
Add an agent run trace view accessible by clicking "View run" on any byline in the Recent Bylines table. It should look like a terminal output but structured: each of the 5 agents (Strategist, LinkedIn Writer, X Writer, Reddit Writer, Critic) appears as a collapsible section with: agent name, execution time, token usage, input/output truncated preview, and any flags raised. Style the whole panel with a dark background (not full black — use #1a1a18 or similar), monospace font for the output text, and colored prefixes: green for success, yellow for warnings, red for flags. Show a timeline bar at the top showing the relative time each agent took.
Prompt 08 — KPI cards with sparklines
Redesign the 4 top metric cards (Posts Published, Avg Critic Score, Platforms Live, Reach This Month). Each card should show: the main number in large monospace font, a percentage change vs last month (green if positive, red if negative), and a tiny 8-point sparkline chart showing the weekly trend. Use recharts or tremor for the sparklines. Make the cards respond to hover with a subtle border highlight. Add a 5th card showing "Voice score trend" — the 8.6 score from the current chart as a card rather than a separate full-width chart.
Prompt 09 — empty states
Add proper empty states to every section that can be empty. For Recent Bylines (no posts yet): center a terminal icon, "Log your first milestone" heading, a code block showing `$ byline log "what you shipped"`, and a button "Or type it here →". For Analytics (no data): a grayed-out chart skeleton with "Post something first — analytics appear after your first byline." For Voice Profile (not trained): show a split showing "Generic AI voice" vs "Your voice" with a placeholder, and a CTA to go train it. Make empty states educational, not apologetic.
Prompt 10 — "The Desk" as primary view
Move the draft review experience ("The Desk") from the landing page demo into a full dashboard route at /desk. When a pipeline run completes, the user is automatically redirected here. The Desk has: left panel showing the original milestone text, project context, and the narrative arc (if set); right panel with 4 platform tabs (LinkedIn, X, Reddit, Threads) each showing the draft with inline editing. Below each draft: the critic score (animated counter from 0 to the score), voice match indicator (checkmark or flag), and platform-specific metadata (LinkedIn: estimated reach; Reddit: suggested subreddit; X: character count / thread length). Approve & Ship button in primary red-orange. An Edit in draft button that unlocks the text for direct editing.
Recommended libraries — open source, MIT-licensed
Every library here is battle-tested, React-compatible, and appropriate for a developer-first self-hosted tool. Sorted by impact on perceived quality.
Avatars and icons
boring-avatars
Deterministic SVG avatars from a string. Perfect for project cards. Use `variant="marble"` with your brand colors.
npm
@tabler/icons-react
7000+ outline icons, MIT. Already partially used. Standardize all icons to this. Consistent stroke weight looks much more polished than mixing icon sets.
npm
react-avatar
User initials + color avatars for the user's own profile in the header. Fallback when no photo.
npm
Animation and interaction
framer-motion
For pipeline node animations, counter animations (critic score), draft approval confetti, and page transitions. Use `AnimatePresence` for route transitions.
npm
canvas-confetti
Lightweight (3kb) confetti burst for the "Approve & Ship" button. One-line integration. Disproportionate delight for the effort.
npm
typewriter-effect
For the landing page hero and potentially a welcome message on first login. Already feels native to the Byline brand.
npm
Charts and data
recharts
Already used for the posts/week and voice score charts. Extend with sparklines using `LineChart` with minimal axes. Add `ResponsiveContainer` to every chart for proper sizing.
npm
react-calendar-heatmap
GitHub-style activity heatmap showing posting frequency over time. A natural addition to the analytics section for "build in public" context.
npm
Developer-first UI
xterm.js
For the agent run trace view. Renders a real terminal-style output. Has a React wrapper `@xterm/xterm`. Use the `FitAddon` for responsive sizing.
npm
cmdk
The command palette library behind Linear's ⌘K menu. Shadcn/ui ships a cmdk wrapper. Handles fuzzy search, keyboard nav, grouping. 20-minute integration for a massive UX upgrade.
npm
shiki
Syntax highlighting for the CLI examples, env variables, and any code snippets in the dashboard or docs. Theme it to match your dark/light mode.
npm
Voice and audio
@xenova/transformers
Run Whisper locally in the browser via WebAssembly. No server needed for voice-to-text. Perfect for the voice note quick-capture button — privacy-first since audio never leaves the device.
npm
wavesurfer.js
Waveform visualization for recorded voice notes in the milestone log. Lets users play back before submitting. Small but builds trust in voice capture quality.
