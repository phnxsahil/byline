Questions

Frequently Answered

How does the project context memory work?

Dispatch stores metadata about your projects (such as core tech stack, milestone history, target links, and previous logs) in a vector store. When you capture a new dispatch, our LangGraph agent retrieves this context so it can weave factual details into your drafts rather than generating generic tech summaries.

Is my data shared with public AI models?

No. When self-hosting, all data stays local to your server container. In the Pro Cloud tier, we use zero-data-retention APIs to call LLM providers. Your voice blueprint profile is privately locked to your user account and never used to train public base models.

Can I self-host this on my local machine?

Yes, Dispatch is 100% open-source under the MIT license. You can spin up the full LangGraph worker and the dashboard server using Docker Compose in seconds. See our Community card deploy guide.

How does the AI Critic score drafts?

The AI Critic runs an audit prompt sequence that acts like an experienced publisher. It scores drafts based on factual clarity, metric inclusion, lack of hyperbole, and suitability for the target platform. You can toggle between Gentle, Harsh, and Brutal feedback modes to match your editing needs.

Docker Self-Hosting Guide
Spin up the local containerized stack in under two minutes.
version: '3.8'

services:
  dispatch-api:
    image: ghcr.io/dispatch/dispatch-api:latest
    ports:
      - "8000:8000"
    environment:
      - DATABASE_URL=postgresql://postgres:secret@db:5432/dispatch
      - LLM_PROVIDER=openai
      - OPENAI_API_KEY=${OPENAI_API_KEY}
    depends_on:
      - db

  db:
    image: postgres:15-alpine
    environment:
      - POSTGRES_PASSWORD=secret
      - POSTGRES_DB=dispatch
    volumes:
      - pgdata:/var/lib/postgresql/data
